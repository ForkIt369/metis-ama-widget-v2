Typography

Typography is a critical aspect of the design system that ensures readability and visual hierarchy.

Heading Example (1.5rem)

Body text example for general content and explanations. (1rem)

Typography Guidelines

  • Font Family: 'Inter', 'Segoe UI', Arial, sans-serif
  • Font Sizes:
    • Heading: 1.5rem (24px)
    • Body: 1rem (16px)
    • Caption/Label: 0.875rem (14px)
  • Line Height: 1.5
  • Letter Spacing: 0.01em
  • Usage: Limit to 2-3 sizes per screen, maintain consistent spacing.

Button States

Always implement all button states for proper user feedback.

  • Default: Primary color, white text
  • Hover: Slightly lighter primary with gradient, shadow
  • Active: Darker primary, inset shadow
  • Disabled: Muted color, 50% opacity
  • Focus: Outline with accent color

RoBIT Character

RoBIT avatars add personality and emotional context to the interface. Choose the appropriate avatar based on the interaction context.

Welcome

Use for onboarding and greeting users

Celebration

Use when congratulating users on achievements

Guidance

Use when providing tips or help information

Starstruck

Use when highlighting exciting features or opportunities

Surprised

Use when presenting unexpected results or insights

RoBIT Usage Guidelines

  • Use the character to add personality to the interface
  • Position RoBIT in a non-intrusive location
  • Display contextual speech bubbles on hover
  • Keep animation subtle and not distracting
  • Choose the appropriate emotional state based on context
  • Maintain consistent sizing and quality across different states
Accessibility Note: When using RoBIT avatars, ensure they're accompanied by proper text descriptions. RoBIT should enhance the experience but not be the sole carrier of important information.

Implementation Guidelines

Follow these guidelines to maintain consistency across components.

Best Practices

  • Accessibility: Maintain contrast ratio ≥ 4.5:1 for text. Ensure keyboard navigation for all controls. Use semantic HTML and ARIA labels.
  • Performance: Use SVG icons, optimize images. Bundle/minify assets. Lazy load heavy content.
  • Responsiveness: Design mobile-first, ensure components adapt to all screen sizes.
  • Consistency: Use the provided CSS variables and component patterns throughout your application.

How to Use This Component Library

To apply these components to your projects:

  1. Copy the CSS variables and base styles to your stylesheet
  2. Follow the component patterns and structure shown in this guide
  3. Reference the design system documentation for guidance on use cases and implementations
  4. Use the utility classes to maintain consistent spacing and layout
  5. Test components across different screen sizes to ensure responsiveness
Note: This component library is maintained as the single source of truth for design decisions. Update it with every major UI change and reference it in all new component development.
